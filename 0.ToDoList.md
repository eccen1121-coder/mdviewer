# MDViewer - ToDoList

## 앱 업데이트 방식

### 수동 업데이트 (현재 방식)

코드 수정 후 직접 빌드하여 .dmg 파일을 재배포하는 방식입니다.

**업데이트 절차:**

1. `src/` 폴더 내 코드 수정 (main.js, renderer.js, styles.css 등)
2. `package.json`에서 버전 번호 올리기 (예: `1.0.0` → `1.0.1`)
3. 터미널에서 빌드 실행:
   ```bash
   cd /Volumes/TAEHO9_02/5. Product/1. MDViewer
   npm run build
   ```
4. `dist/` 폴더에 생성된 `.dmg` 파일을 사용자에게 전달
   - `MDViewer-x.x.x-arm64.dmg` — Apple Silicon (M1/M2/M3)
   - `MDViewer-x.x.x.dmg` — Intel x64

> **적합한 경우:** 개인 사용 또는 소규모 배포 시 충분한 방식입니다.

---

### 자동 업데이트 시스템 (향후 구현 예정)

앱 실행 시 서버에서 최신 버전을 확인하고, 업데이트가 있으면 앱 내에서 알림을 표시하는 방식입니다.

**구현 구성 요소:**

| 구성 요소 | 내용 |
|-----------|------|
| 버전 정보 서버 | GitHub 공개 저장소에 `version.json` 파일 호스팅 |
| 앱 내 체크 로직 | 실행 시 또는 주기적으로 최신 버전 조회 |
| 알림 UI | "새 버전이 있습니다" 배너 또는 다이얼로그 표시 |
| 다운로드 링크 | 클릭 시 GitHub Releases 다운로드 페이지로 이동 |

**구현 체크리스트:**

- [ ] GitHub 공개 저장소에 `version.json` 파일 업로드
  ```json
  { "version": "1.0.1", "downloadUrl": "https://github.com/.../releases" }
  ```
- [ ] 앱 실행 시 GitHub에서 최신 버전 번호 자동 확인
- [ ] 현재 버전과 비교하여 새 버전이 있을 경우 앱 내 알림 표시
- [ ] 알림 클릭 시 다운로드 페이지(GitHub Releases 등)로 연결

> **참고:** 자동 다운로드 및 설치(`electron-updater`)는 Apple 코드 서명(Apple Developer 계정, 연 $99)이 필요하므로 우선 버전 체크 알림 방식으로 구현 예정.
